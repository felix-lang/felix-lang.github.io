1 Basics

Felix is a unique language in that, unlike other languages, it does not have any primitive types.

Instead, Felix provides machinery to lift types from C++ code. This means Felix is really a C++ meta-programming language.

I will show the basics here but first a caveat: we have to use weird names for things to avoid conflicts with the Felix standard library. 

examples/embedc/ex1.flx

  type cplx = "::std::complex<double>" 
    requires header '#include <complex>'
  ;
  ctor cplx : double * double = 
    "::std::complex<double>($1,$2)"
  ;
  fun + : cplx * cplx -> cplx = "$1+$2";
  fun - : cplx * cplx -> cplx = "$1-$2";
  fun * : cplx * cplx -> cplx = "$1*$2";
  fun / : cplx * cplx -> cplx = "$1/$2";
  fun == : cplx * cplx -> bool = "$1==$2";
  fun real: cplx -> double = "$1.real()";
  fun imag: cplx -> double = "$1.imag()";
  fun abs: cplx -> double = "$1.abs()";
  fun arg: cplx -> double = "$1.arg()";
  fun norm: cplx -> double = "$1.norm()";
  fun conj: cplx -> double = "$1.conj()";
  
  typedef polarbear = cplx;
  ctor polarbear: double * double = "::std::polar<double>($1,$2)";
  
  fun str (x: cplx) => x.real.str + "+" + x.imag.str + "i";
  
  // test
  var x = cplx (1.0,2.0);
  var y = polarbear (1.0,0.5);
  println$ (x+y).str;
  println$ (x == y).str;

1.1 Type Classes

Let us try an experiment: 

examples/embedc/ex1.flx

  println$ (x,y);

If you run it at this point without the rest of the tutorial, this happens: 

CLIENT ERROR
[flx_frontend/flx_typeclass.ml:752: E367] [Cannotmatch] Cannot instantiate virtual str<17781>[Nominal[] cplx]
In /Users/skaller/felix/src/packages/core_type_constructors.fdoc: line 693, cols 4 to 59
692: instance[T] Str[T*T] {
693:    fun str (t1:T, t2:T) => "("+str t1 + ", " + str t2+")";
        ********************************************************
694: }

Its not very helpful. But here is a fix: 

examples/embedc/ex1.flx

  instance Str[cplx] {
    fun str (x: cplx) => x.real.str + "+" + x.imag.str + "i";
  }

and now we get this: 

(1+2i, 0.877583+0.479426i)

To understand what's going on you need to see some of the Standard Library: 

  class Str [T] {
    virtual fun str: T -> string;
  }

By instantiating the virtual function in an instance the definition is available universally.

2 Another type class

Since the last type class was a bit easy here's a slightly harder one.. only slightly harder I promise! 

  // equality: technically, equivalence relation
  class Eq[t] {
    virtual fun == : t * t -> bool;
    virtual fun != (x:t,y:t):bool => not (x == y);
  
    axiom reflex(x:t): x == x;
    axiom sym(x:t, y:t): (x == y) == (y == x);
    axiom trans(x:t, y:t, z:t): x == y and y == z implies x == z;
  
    fun eq(x:t, y:t)=> x == y;
    fun ne(x:t, y:t)=> x != y;
    fun \(\ne\)(x:t, y:t)=> x != y;
    fun \(\neq\)(x:t, y:t)=> x != y;
  }

This is a Felix type class . The design is modelled after Haskell. It has:

  1. A name Eq
  2. A type parameter t
  3. An undefined virtual function for equality
  4. An virtual function for inequality with a default definition
  5. A set of axiom which help to define the semantics
  6. A collection of derived functions all of which are trivial renamings

You have already seen the syntax for instantiating virtuals by setting the type variable t to the type cplx. The inquality can be defined as well but if not, as in this case, is derived from the euqliaty.

The derived functions are available with the class also, but cannot be defined in an instance.
